# code-review-update

代码审查后自动修复技能，根据 code-review 的审查建议自动修复代码问题。

## 功能描述

分析 code-review 技能的输出结果，自动识别可以修复的问题，并进行代码修改。对于需要确认的修改，会询问用户后再执行。

## 目录结构

```
code-review-update/
└── SKILL.md               # 技能配置文件
```

## 修复策略

根据问题的严重程度分类处理：

### P0 - 必须修复

- 安全漏洞
- 严重 bug
- 导致系统崩溃的问题

**处理方式:** 自动修复

### P1 - 应该修复

- 性能问题
- 逻辑错误
- 潜在的运行时错误

**处理方式:** 自动修复

### P2 - 建议修复

- 代码规范问题
- 可读性问题
- 命名规范

**处理方式:** 询问用户后修复

## 自动修复类型

以下类型的问题会自动修复：

- 简单的代码规范问题
- 明显的逻辑错误
- 简单的性能优化
- 未使用的变量和导入
- 缺失的异常处理（简单情况）

## 需要用户确认

以下情况需要用户确认：

- 业务逻辑变更
- API 接口变更
- 数据结构变更
- 可能影响现有功能的修改
- 复杂的重构

## 使用方法

### 基本用法

```
/code-review-update
```

该命令会读取最近的 code-review 输出，分析可修复的问题，并开始自动修复流程。

### 工作流程

1. 读取 code-review 的输出
2. 解析问题和修复建议
3. 按优先级排序（P0 → P1 → P2）
4. 逐个处理问题：
   - 自动修复类：直接修改代码
   - 需确认类：询问用户后处理
5. 生成修复报告

## 输出报告

修复完成后会输出：

- 修复的问题数量
- 每个问题的修复详情
- 需要手动处理的问题列表
- 建议的后续操作
